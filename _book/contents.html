1 Building distributed services with Go

2 Overview.

I'd like to write a book to help entry-mid level Go programmers take the next step after learning the language and learn how to build projects. There is a lot of material available on learning the language but little on using it to build real-world projects, which we're all trying to do as programmers. The book will be organized around building and deploying a distributed commit log, providing the opportunity to build a real-world project using many of the ideas and tools Go programmers use: how to layout a project, configuration, use protobufs, write a server and client, test and monitor a service, and so on. A distributed commit log will also provide the opporunity for experienced Go programmers to learn how distributed services work and how to build their own, like Kafka, Consul, Etcd.
